📚 Case Studies

Windows Domain Administration

Real-world scenarios and solutions for Windows operating systems and server administration challenges encountered during my BSIT-S-2A studies.

PART 1: DOMAIN CONTROLLER

Configure the server as a Domain Controller using Active Directory Domain Services. Set the domain name to winvault.local, verify DNS integration, and make sure client workstations can join the domain successfully.
Case Study Link: https://youtu.be/U2XYNXHr7kc

PART 2: ADDING USERS

Create organizational units for every academic technology section and administrative office, then add domain users using real names. Apply account controls including workstation logon assignment, password expiration every 30 days, and logon hours: academic users (7:30 AM-8:45 AM) and administrative users (7:00 AM-7:00 PM).
Case Study Link: https://youtu.be/cZs77Q7alvw

PART 3: DHCP SERVER

Configure DHCP scopes for 172.31.0.0/16 and 10.0.0.0/8, set the router as a valid gateway, and use the DHCP server IP as DNS while adding Google DNS and OpenDNS as additional resolvers. Set lease time to 8 hours and exclude the first 30 and last 30 valid IP addresses from distribution.
Case Study Link: https://youtu.be/ngQXdd0AvTM

PART 4: FILE SHARING

Create department and section public folders with read/write access for their own users only, add a university-wide public folder for all domain users, and create an administrator-only installer folder. Enforce 1 GB quota per folder, apply file-type restrictions per section requirements, and map the proper shared drives on each client workstation.
Case Study Link: https://youtu.be/C53gVyXxE4Y

PART 5: RAID

Create and validate the required RAID configurations: striped, mirrored, spanned, and RAID-5. Document how each setup affects performance, fault tolerance, and usable storage in the server environment.
Case Study Link: https://youtu.be/IAybl3qPDzU

PART 6: WEB SERVER

Install and configure Internet Information Services (IIS) for web hosting. Set up multiple websites, configure application pools, implement SSL certificates, and establish security policies for web applications and FTP services.
Case Study Link: https://youtu.be/example-webserver

PART 7: GROUP POLICY

Implement Group Policy Objects (GPOs) to manage user and computer settings across the domain. Configure security policies, software deployment, desktop restrictions, and administrative templates to standardize the computing environment.
Case Study Link: https://youtu.be/example-gpo

PART 8: WINDOWS DEPLOYMENT SERVICES

Configure Windows Deployment Services (WDS) for network-based OS installation. Set up deployment images, create boot images, configure PXE boot, and implement automated installation procedures for Windows workstations and servers.
Case Study Link: https://youtu.be/example-wds
